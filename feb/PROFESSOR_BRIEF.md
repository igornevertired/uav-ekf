## Вводные данные и что делает система (для преподавателя)

Этот документ описывает:
- **векторы состояния** и **управления**;
- **что именно идентифицируется**;
- **какие ошибки/шумы моделируются** (с численными значениями);
- как устроены **режим 2** и **режим 3** (идеальная навигация vs навигационное решение БИНС+ГНСС).

---

## 1) Модель объекта (продольная динамика)

### Состояние истинной динамики (истина в симуляторе)

Формула:

    x_true = [ V, alpha, q, theta ]^T

- V — воздушная скорость, м/с  
- alpha — угол атаки, рад  
- q — угловая скорость тангажа, рад/с  
- theta — угол тангажа, рад

**Файл:** `models/aircraft_dynamics.py`

### Управление

Формула:

    u = delta_e

- delta_e — отклонение руля высоты (рад)

**Файл:** `control/excitation.py` — профили возмущений (ступени + синусы).

---

## 2) Что идентифицируется (нижний уровень: Parameter EKF)

### Вектор состояния Parameter EKF (расширенный)

Формула:

    x_id = [ V, alpha, q, theta, CL_alpha, CL_delta_e, Cm_alpha, Cm_qbar, Cm_delta_e ]^T

Размерность: **9**.

### Идентифицируемые параметры (5 шт.)

Формула:

    p = [ CL_alpha, CL_delta_e, Cm_alpha, Cm_qbar, Cm_delta_e ]^T

Это производные (аэродинамические коэффициенты), которые в симуляторе считаются **константами** и должны быть оценены EKF.

**Файл:** `filters/parameter_ekf.py`

---

## 3) Измерения для идентификации (что подаётся в update нижнего EKF)

### Вектор измерений

Формула:

    y = [ V_m, alpha_m, q_m, theta_m, nz_m ]^T

Размерность: **5**.

**Файл:** `sensors/sensor_models.py`, метод `generate_sensor_data(...)`.

### Как считается перегрузка nz

Формула:

    nz = (L*cos(alpha) + D*sin(alpha)) / (m*g)

где L и D берутся из аэродинамической модели.

**Файл:** `models/aircraft_dynamics.py`, метод `compute_load_factor(...)`.

---

## 4) Моделируемые ошибки измерений (датчики идентификации)

Это ошибки **внутри** `SensorModels`.

### 4.1 Шум (СКО)

Задано в `SensorModels.noise_levels`:

- **скорость**: sigma_V = 0.5 м/с  
- **угол атаки**: sigma_alpha = 0.001 рад (~0.06 deg)  
- **угловая скорость**: sigma_q = 0.001 рад/с (~0.06 deg/s)  
- **угол тангажа**: sigma_theta = 0.001 рад (~0.06 deg)  
- **перегрузка**: sigma_nz = 0.005 g

### 4.2 Смещения (bias)

Задано в `SensorModels.biases`. В текущей конфигурации все bias = 0.0, но модель поддерживает их установку.

### 4.3 Динамическая ошибка датчика угла атаки

Формула:

    alpha_m = alpha + d_aoa*(q/V) + b_alpha + nu_alpha

где `d_aoa = AircraftParams.AOA_SENSOR_DYNAMIC_ERROR`.

Примечание: для БПЛА в текущей конфигурации задано d_aoa = 0.20 м (оценка), чтобы динамическая
ошибка AoA не доминировала и не сдвигала идентификацию CL_delta_e.

## 5) Навигационный комплекс (верхний уровень: INS + GNSS + Navigation EKF)

### 5.1 Strapdown INS (БИНС)

INS распространяет состояние позиции/скорости/ориентации и моделирует ошибки датчиков:

- шум гироскопа (ARW),
- шум акселерометра (VRW),
- смещения гироскопа и акселерометра,
- случайное блуждание (random walk) смещений.

**Файл:** `models/ins_model.py`, метод `propagate(..., include_errors=True)`.

### 5.2 Численные параметры ошибок INS (по классам точности)

Берутся из `config/navigation_params.py`:

**Commercial INS:**
- gyro_bias_stability = 10 deg/hr
- gyro_arw = 0.5 deg/√hr
- accel_bias_stability = 1.0 mg
- accel_vrw = 0.05 m/s/√hr
- update_rate = 100 Hz

**Tactical INS:**
- gyro_bias_stability = 1 deg/hr
- gyro_arw = 0.1 deg/√hr
- accel_bias_stability = 0.1 mg
- accel_vrw = 0.01 m/s/√hr
- update_rate = 200 Hz

**Navigation INS:**
- gyro_bias_stability = 0.01 deg/hr
- gyro_arw = 0.001 deg/√hr
- accel_bias_stability = 0.01 mg
- accel_vrw = 0.001 m/s/√hr
- update_rate = 400 Hz

### 5.3 GNSS (ГНСС)

GNSS выдаёт позицию и скорость с белым шумом, с фиксированной частотой.

**Файл:** `models/gnss_model.py`, метод `GNSSReceiver.get_measurement(...)`.

**Commercial GNSS:**
- position_std = 7.5 м
- velocity_std = 0.15 м/с
- update_rate = 5 Hz

**Tactical GNSS:**
- position_std = 2.0 м
- velocity_std = 0.05 м/с
- update_rate = 10 Hz

**Navigation GNSS:**
- position_std = 0.5 м
- velocity_std = 0.01 м/с
- update_rate = 20 Hz

### 5.4 Navigation EKF (комплексирование)

Navigation EKF оценивает **ошибки** БИНС (15 состояний) и корректирует их по GNSS.

Формула:

    x_nav = [ dpos(3), dvel(3), datt(3), gyro_bias(3), accel_bias(3) ]^T

**Файл:** `filters/navigation_ekf.py`

## 6) Чем отличаются режим 2 и режим 3 (ключевой смысл эксперимента)

### Режим 2 (идеализированный)

- Вектор измерений y строится из **истинного состояния динамики** x_true + шум/смещения датчиков идентификации.  
- Навигационные ошибки (INS/GNSS) не влияют на идентификацию.

Цель: показать «верхнюю границу» качества идентификации при идеальной навигации.

### Режим 3 (ближе к реальности)

- Вектор измерений y строится из **навигационного решения** (INS+GNSS+Navigation EKF).  
- Дополнительный шум датчиков идентификации **не добавляется** (`add_noise=False`), потому что погрешности уже присутствуют в навигационном решении.

Доработка режима 3: скорость переводится в связанную СК; V и alpha берутся из компонент в теле:

Формулы:

    V = sqrt(vx_body^2 + vz_body^2)
    alpha = atan2(vz_body, vx_body)
Также опционально применяется экспоненциальное сглаживание (EMA) навигационных оценок.

Цель: исследовать, как качество навигации влияет на точность идентификации.

## 8) Важное допущение для корректной идентификации производных

Идентифицируемые величины CL_alpha, Cm_alpha, Cm_qbar и т.п. — это **производные линейной модели**,
которые корректно интерпретируются в окрестности режима (trim) и при малых углах атаки.

Поэтому в симуляции специально обеспечивается работа в **линейной области**:

- ограничение истинного угла атаки: |alpha| <= ALPHA_MAX_SAFE (12 deg)
- умеренное возбуждение руля высоты (профиль enhanced с небольшими ступеньками и синусоидами)

Это стандартная практика: если постоянно выходить в срыв/сильные нелинейности, EKF неизбежно будет
пытаться «объяснить» нелинейность постоянными параметрами и оценки станут смещёнными.
