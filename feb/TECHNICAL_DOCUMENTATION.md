# КОМПЛЕКСНАЯ СИСТЕМА ИДЕНТИФИКАЦИИ АЭРОДИНАМИЧЕСКИХ ПАРАМЕТРОВ С НАВИГАЦИОННЫМ КОМПЛЕКСОМ БИНС+ГНСС

## Полная техническая документация проекта

**Объект исследования:** Cessna 172R Skyhawk  
**Метод:** Расширенный фильтр Калмана (EKF)

## АННОТАЦИЯ

Разработана комплексная система для **онлайн-идентификации аэродинамических параметров** самолёта Cessna 172R Skyhawk с использованием метода расширенного фильтра Калмана и интегрированного навигационного комплекса БИНС+ГНСС.

**Ключевые особенности:**
- Реальные данные самолёта из сертифицированных источников (NASA, Cessna POH)
- Полная нелинейная аэродинамическая модель
- Учёт сжимаемости, срыва потока, нестационарных эффектов
- Точность идентификации: < 2%
- Эффективность комплексирования навигации: 72%

### 2.2 Принципы проектирования

#### SOLID принципы:

1. **Single Responsibility**: каждый класс отвечает за одну задачу
   - `AerodynamicCoefficients` → только аэродинамика
   - `AircraftDynamics` → только динамика
   - `StrapdownINS` → только БИНС механизация

2. **Open/Closed**: открыто для расширения, закрыто для модификации
   - `EKFBase` → базовый класс для всех EKF
   - Легко добавить новые фильтры без изменения базового

3. **Liskov Substitution**: любой наследник взаимозаменяем
   - `ParameterEKF` и `NavigationEKF` → оба работают через `EKFBase`

4. **Interface Segregation**: клиенты не зависят от неиспользуемых интерфейсов
   - Каждый модуль имеет минимальный API

5. **Dependency Inversion**: зависимость от абстракций, не от конкретных классов
   - `AircraftDynamics` использует `AerodynamicCoefficients` через композицию

#### DRY (Don't Repeat Yourself):

- Общие параметры → `AircraftParams`, `NavigationParams`
- Общая логика EKF → `EKFBase`
- Утилиты → вынесены в отдельные модули

## 3. МАТЕМАТИЧЕСКИЕ МОДЕЛИ {#математика}

### 3.1 Динамика самолета

#### Система координат:
- **Связанная СК** (body frame): ось X вперед, Z вниз
- **Навигационная СК** (NED): North-East-Down

#### Состояние:
```
x = [V, α, q, θ]ᵀ
```
где:
- `V` — воздушная скорость (м/с)
- `α` — угол атаки (рад)
- `q` — угловая скорость тангажа (рад/с)
- `θ` — угол тангажа (рад)

#### Уравнения движения:

```
V̇ = (T·cos(α) - D - m·g·sin(θ-α)) / m

α̇ = (-L + m·g·cos(θ-α)) / (m·V) + q

q̇ = M / Iyy

θ̇ = q
```

где:
- `T` — тяга двигателя (Н)
- `L` — подъемная сила (Н)
- `D` — сила лобового сопротивления (Н)
- `M` — момент тангажа (Н·м)
- `m` — масса (кг)
- `Iyy` — момент инерции относительно оси Y (кг·м²)
- `g` — ускорение свободного падения (м/с²)

### 3.2 Аэродинамические коэффициенты

#### Модель:

```
CL = CL₀ + CLα·α + CLδe·δe
CD = CD₀ + CDα·α
Cm = Cm₀ + Cmα·α + Cmq̄·(q·c̄)/(2V) + Cmδe·δe
```

#### Силы и момент:

```
q_dyn = 0.5 · ρ · V²
L = q_dyn · S · CL
D = q_dyn · S · CD
M = q_dyn · S · c̄ · Cm
```

где:
- `ρ` — плотность воздуха (кг/м³)
- `S` — площадь крыла (м²)
- `c̄` — средняя аэродинамическая хорда (м)
- `δe` — отклонение руля высоты (рад)

#### Идентифицируемые параметры:

```
p = [CLα, CLδe, Cmα, Cmq̄, Cmδe]ᵀ
```

**Истинные значения:**
- `CLα = 3.45` рад⁻¹
- `CLδe = 0.4` рад⁻¹
- `Cmα = -0.41` рад⁻¹
- `Cmq̄ = -4.3` рад⁻¹
- `Cmδe = -0.6` рад⁻¹

### 3.3 Бесплатформенная ИНС (БИНС)

#### Механизация БИНС:

**Состояние:**
```
x_INS = [p, v, q, b_g, b_a]ᵀ
```
где:
- `p` — позиция [x, y, z]ᵀ в NED (м)
- `v` — скорость [vₓ, vᵧ, vᵧ]ᵀ в NED (м/с)
- `q` — кватернион ориентации [q₀, q₁, q₂, q₃]ᵀ
- `b_g` — смещения гироскопов [bₓ, bᵧ, bᵧ]ᵀ (рад/с)
- `b_a` — смещения акселерометров [bₓ, bᵧ, bᵧ]ᵀ (м/с²)

**Уравнения обновления:**

1. **Ориентация** (кватернионная кинематика):
```
q̇ = 0.5 · Ω(ω) · q

где Ω(ω) = [  0   -ωₓ  -ωᵧ  -ωᵧ ]
           [ ωₓ    0    ωᵧ  -ωᵧ ]
           [ ωᵧ  -ωᵧ    0    ωₓ ]
           [ ωᵧ   ωᵧ  -ωₓ    0  ]
```

2. **Скорость**:
```
v̇ = C_b→n · f_b + g_n
```
где:
- `C_b→n` — матрица направляющих косинусов (DCM) из кватерниона
- `f_b` — удельная сила в связанной СК (м/с²)
- `g_n` — вектор гравитации в навигационной СК [0, 0, -g]ᵀ

3. **Позиция**:
```
ṗ = v
```

4. **Смещения датчиков** (случайное блуждание):
```
ḃ_g = w_g  (белый шум с PSD = σ²_bg)
ḃ_a = w_a  (белый шум с PSD = σ²_ba)
```

#### Модель ошибок датчиков:

**Гироскопы:**
```
ω_meas = ω_true + b_g + η_g
```
где:
- `η_g` ~ N(0, σ²_ARW) — Angle Random Walk (ARW)
- `σ_ARW` зависит от класса точности

**Акселерометры:**
```
f_meas = f_true + b_a + η_a
```
где:
- `η_a` ~ N(0, σ²_VRW) — Velocity Random Walk (VRW)
- `σ_VRW` зависит от класса точности

#### Классы точности БИНС:

| Класс | Gyro bias stability | ARW | Accel bias stability | VRW |
|-------|---------------------|-----|----------------------|-----|
| **Commercial** | 10 °/hr | 0.1 °/√hr | 1 mg | 0.05 m/s/√hr |
| **Tactical** | 1 °/hr | 0.05 °/√hr | 0.5 mg | 0.01 m/s/√hr |
| **Navigation** | 0.01 °/hr | 0.001 °/√hr | 25 μg | 0.001 m/s/√hr |

#### Дрейф БИНС:

Без коррекции ошибка позиции растет **квадратично**:
```
δp(t) ≈ 0.5 · b_a · t²
```

Для Commercial класса с `b_a ≈ 10⁻² m/s²`:
```
t = 300 сек → δp ≈ 450 км  (катастрофа!)
```

### 3.4 Глобальная навигационная спутниковая система (ГНСС)

#### Модель измерений:

```
z_GNSS = [p_GNSS, v_GNSS]ᵀ
```

**Ошибки:**
```
p_GNSS = p_true + η_p,  η_p ~ N(0, σ²_pos)
v_GNSS = v_true + η_v,  η_v ~ N(0, σ²_vel)
```

#### Параметры точности:

| Класс | σ_pos | σ_vel | Частота |
|-------|-------|-------|---------|
| **Commercial** | 1.0 м | 0.01 м/с | 10 Гц |
| **Tactical** | 0.5 м | 0.005 м/с | 10 Гц |
| **Navigation** | 0.1 м | 0.001 м/с | 10 Гц |

**Ключевое свойство:** ошибка ГНСС **не растет со временем** (независима от прошлых измерений).

### 3.5 Parameter EKF (идентификация параметров)

#### Расширенное состояние:

```
x_aug = [V, α, q, θ, CLα, CLδe, Cmα, Cmq̄, Cmδe]ᵀ  (9 × 1)
```

#### Динамика:

```
ẋ_state = f(x_state, u, p)  // параметры в уравнениях
ṗ = 0                        // параметры постоянны
```

#### Измерения:

```
y = [V_meas, α_meas, q_meas, θ_meas, nz_meas]ᵀ  (5 × 1)
```

где:
- `α_meas = α + d_aoa · q/V` — динамическая ошибка датчика АОА
- `nz = (L·cos(α) + D·sin(α)) / (m·g)` — нормальная перегрузка

#### Алгоритм EKF:

**Predict:**
```
x̂⁻ₖ = f(x̂⁺ₖ₋₁, uₖ, Δt)
Pₖ⁻ = Fₖ · Pₖ₋₁⁺ · Fₖᵀ + Q
```

где:
```
Fₖ = ∂f/∂x|ₓ=ₓ̂ₖ₋₁  (аналитический или численный якобиан)
```

**Update:**
```
yₖ = измерения
ŷₖ = h(x̂ₖ⁻, uₖ)
νₖ = yₖ - ŷₖ  (инновация)

Sₖ = Hₖ · Pₖ⁻ · Hₖᵀ + R
Kₖ = Pₖ⁻ · Hₖᵀ · Sₖ⁻¹  (усиление Калмана)

x̂ₖ⁺ = x̂ₖ⁻ + Kₖ · νₖ
Pₖ⁺ = (I - Kₖ·Hₖ) · Pₖ⁻ · (I - Kₖ·Hₖ)ᵀ + Kₖ·R·Kₖᵀ  (форма Иосифа)
```

где:
```
Hₖ = ∂h/∂x|ₓ=ₓ̂ₖ⁻
```

**Ограничение параметров:**
```
CLα   ∈ [2.5, 4.5]
CLδe  ∈ [0.2, 0.6]
Cmα   ∈ [-0.6, -0.2]
Cmq̄  ∈ [-6.0, -2.5]
Cmδe  ∈ [-0.8, -0.4]
```

#### Матрицы ковариаций:

**Q (шум процесса):**
```
Q = diag([0.5, 10⁻⁶, 10⁻⁶, 10⁻⁶,     // состояния
          10⁻⁸, 10⁻⁸, 10⁻⁸, 10⁻⁶, 10⁻⁸])  // параметры
```

**R (шум измерений):**
```
R = diag([0.25, 10⁻⁶, 10⁻⁶, 10⁻⁶, 2.5·10⁻⁵])
```

### 3.6 Navigation EKF (комплексирование БИНС+ГНСС)

#### Подход: Error-State EKF

Вместо оценки полного состояния, оцениваем **ошибки** БИНС:

```
δx = [δp, δv, δψ, b_g, b_a]ᵀ  (15 × 1)
```

где:
- `δp` — ошибки позиции (3 × 1)
- `δv` — ошибки скорости (3 × 1)
- `δψ` — ошибки ориентации (малые углы, 3 × 1)
- `b_g` — смещения гироскопов (3 × 1)
- `b_a` — смещения акселерометров (3 × 1)

#### Динамика ошибок:

**Упрощенная линейная модель:**
```
δ̇p = δv
δ̇v = -[f×]·δψ - C_b→n·b_a + w_v
δ̇ψ = -b_g + w_ψ
ḃ_g = w_bg
ḃ_a = w_ba
```

**Матрица F (якобиан):**
```
F = [ I    I·dt   0      0       0    ]  δp
    [ 0     I   -[f×]·dt  0   -C·dt  ]  δv
    [ 0     0     I   -I·dt     0    ]  δψ
    [ 0     0     0      I       0    ]  b_g
    [ 0     0     0      0       I    ]  b_a
```

где `[f×]` — кососимметричная матрица удельной силы.

#### Измерения от ГНСС:

```
y_GNSS = [p_GNSS - p_INS, v_GNSS - v_INS]ᵀ
```

**Матрица H:**
```
H = [ I₃ₓ₃  0₃ₓ₃  0₃ₓ₃  0₃ₓ₃  0₃ₓ₃ ]  // измеряем δp
    [ 0₃ₓ₃  I₃ₓ₃  0₃ₓ₃  0₃ₓ₃  0₃ₓ₃ ]  // измеряем δv
```

#### Коррекция БИНС:

После update EKF применяем коррекцию:

```
p_corr = p_INS + δp
v_corr = v_INS + δv
ψ_corr = ψ_INS + δψ
```

**ВАЖНО:** После коррекции сбрасываем `δp, δv, δψ → 0`, но оставляем `b_g, b_a` (они накапливаются).

#### Матрицы ковариаций:

**Q (увеличена для учета дрейфа):**
```
Q = diag([
    10⁻⁴ · dt², ...,    // позиция (квадратичный рост)
    10⁻⁴ · 50, ...,     // скорость
    10⁻⁶ · 10, ...,     // ориентация
    10⁻¹⁰ · 5, ...,     // bias гироскопов
    10⁻⁸ · 5, ...       // bias акселерометров
])
```

**R (от ГНСС):**
```
R = diag([σ²_pos, σ²_pos, σ²_pos, σ²_vel, σ²_vel, σ²_vel])
```

### 3.7 Профили управления (Persistent Excitation)

Для точной идентификации нужен **богато возбуждающий** сигнал.

#### Enhanced profile (используется по умолчанию):

```python
if t < 14:
    # Ступенчатые маневры ±1-2°
    δe = doublets(t)
else:
    # Многочастотная синусоида
    δe = 1.5·sin(0.5t) + 0.8·sin(1.5t) + 0.3·sin(3.0t)  [градусы]
```

**Почему 2 фазы?**

1. **Ступеньки (0-14 сек)**: быстрая первичная оценка параметров
2. **Синусоиды (14+ сек)**: точная настройка, возбуждение всех мод

Разные частоты возбуждают разные параметры:
- **0.5 рад/с**: `CLα`, `Cmα` (статическая устойчивость)
- **1.5 рад/с**: `CLδe`, `Cmδe` (эффективность управления)
- **3.0 рад/с**: `Cmq̄` (демпфирование)

#### Другие профили:

- **Doublet**: дублеты (±A, затем ∓A)
- **3-2-1-1**: стандартный идентификационный маневр
- **Multisine**: многочастотный синус
- **Chirp**: sweep по частотам

#### Пример: кватернионная механика в БИНС

```python
def _update_attitude(self, omega_body, dt):
    omega_mag = np.linalg.norm(omega_body)
    if omega_mag < 1e-8:
        return
    
    # Кватернион приращения вращения
    half_angle = 0.5 * omega_mag * dt
    axis = omega_body / omega_mag
    delta_q = np.array([
        np.cos(half_angle),
        np.sin(half_angle) * axis[0],
        np.sin(half_angle) * axis[1],
        np.sin(half_angle) * axis[2]
    ])
    
    # Умножение кватернионов
    self.quaternion = self._quaternion_multiply(delta_q, self.quaternion)
    self.quaternion /= np.linalg.norm(self.quaternion)
```

```python
while t < T:
    # 1. Управление
    delta_e = control.get_control(t, profile_type)
    
    # 2. Истинная динамика самолета
    x_true = integrate(aircraft_dynamics, x_true, delta_e, dt)
    
    # 3. БИНС + ГНСС + Navigation EKF
    if use_navigation:
        ins_state = ins.propagate(dt, specific_force, angular_rate)
        gnss_meas = gnss.get_measurement(true_pos, true_vel, dt)
        nav_ekf.predict()
        nav_ekf.update_with_gnss(gnss_meas, ins_state)
        corrected = nav_ekf.get_corrected_state(ins_state)
    
    # 4. Измерения датчиков для идентификации
    y_meas = sensors.generate_sensor_data(x_true, delta_e)
    
    # 5. Parameter EKF
    param_ekf.predict(delta_e)
    param_ekf.update(y_meas, delta_e)
    
    # 6. Визуализация
    if visualize:
        plotter.update(param_ekf, x_true, t)
    
    # 7. Сохранение в историю
    history['ins_raw'].append(ins_state)
    history['gnss_measurement'].append(gnss_meas)
    history['nav_corrected'].append(corrected)
    
    t += dt
```
