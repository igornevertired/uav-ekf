# Модели БИНС и ГНСС

В репозитории две реализации: **бесплатформенная ИНС** (`src/bins.py`) и **приёмник ГНСС** (`src/gnss.py`). Параметры ошибок задаются в JSON в конце этого файла и загружаются через `src.params.load_params()`.

---

## БИНС (`StrapdownINS`)

### Какие приборы

| Прибор | Что выдаёт | Роль в коде |
|--------|------------|-------------|
| **Трёхосевой гироскоп** | Угловая скорость тела в связанной СК, рад/с | Задаётся снаружи как «истина» `angular_rate_body`; на измерение накладываются смещение и шум (см. ниже). По ним интегрируется кватернион ориентации. |
| **Трёхосевой акселерометр** | Удельная сила в связанной СК, м/с² | Задаётся снаружи как `specific_force_body` **или** при `level_flight_ned=True` подставляется трим `Rᵀ(−g)` после шага ориентации. На измерение — смещение и шум. |

### Какие отклонения моделируются

| Величина | Параметр в JSON | Смысл |
|----------|-----------------|--------|
| Стабильность смещения гироскопа | `gyro_bias_stability_deg_hr` | Порядок СКО **начального** смещения по осям (°/ч → в коде рад/с). |
| ARW гироскопа | `gyro_arw_deg_sqrt_hr` | Angle random walk: дисперсия белого шума на измеренной ω связана с ARW (°/√ч). |
| Стабильность смещения акселерометра | `accel_bias_stability_mg` | СКО **начального** смещения (мg → м/с²). |
| VRW акселерометра | `accel_vrw_m_s_sqrt_hr` | Velocity random walk: шум на удельной силе (м/с/√ч). |
| Частота (симуляция) | `update_rate_hz` | Используется как метаданные набора; шаг интегрирования задаётся вызывающим кодом (`dt`). |

Дополнительно в коде: **упрощённое случайное блуждание** смещений гиро/акселя на шаге (дрейф bias), коэффициент 0.1 к масштабу СКО смещения × √dt.

Система координат навигации: **NED**; ускорение в NED: `R·f_изм + g`, где `g = [0, 0, g]` (вниз).

Дополнительно в коде: ``body_to_ned_dcm()``, ``get_state()``, ``reset(...)`` — матрица направляющих косинусов, снимок состояния, сброс с пересэмплированием смещений.

## ГНСС (`GNSSReceiver`)

### Какой прибор

Один абстрактный **приёмник**, выдающий готовые оценки **позиции** и **скорости** в той же СК (NED), что и переданная «истина».

### Какие отклонения

| Величина | Параметр в JSON | Смысл |
|----------|-----------------|--------|
| Шум позиции | `position_std_m` | СКО независимого гауссова шума по каждой из осей x, y, z (м). |
| Шум скорости | `velocity_std_m_s` | То же для компонент скорости (м/с). |
| Частота решений | `update_rate_hz` | Не чаще одного нового измерения за `1/частота` секунд. |

Не моделируются в базовом классе: ионосфера, мультипуть, видимость спутников, DOP. Для грубого учёта дополнительных ошибок по позиции есть класс ``GNSSSimulator`` в ``src/gnss.py`` (опционально мультипуть / вертикальная ионосфера).

Дополнительные методы приёмника: ``is_measurement_ready``, ``get_statistics``, ``get_covariance_matrices``, ``simulate_signal_loss``, ``print_info``.

---

## Параметры в коде (JSON)

```json
{
  "gravity_m_s2": 9.80665,
  "simulation": {
    "dt_s": 0.02,
    "duration_s": 60.0,
    "default_seed": 42
  },
  "ins": {
    "gyro_bias_stability_deg_hr": 0.02,
    "gyro_arw_deg_sqrt_hr": 0.002,
    "accel_bias_stability_mg": 0.02,
    "accel_vrw_m_s_sqrt_hr": 0.002,
    "update_rate_hz": 200.0
  },
  "gnss": {
    "position_std_m": 0.25,
    "velocity_std_m_s": 0.02,
    "update_rate_hz": 20.0
  }
}
```

Поле `simulation` используется только внешними скриптами; ядро БИНС/ГНСС его не читает.

## Сводка: приборы и отклонения (числа из JSON выше)

| Подсистема | Прибор / выход | Что задаёт отклонение | Значение в JSON (пример) |
|------------|----------------|------------------------|---------------------------|
| БИНС | Гироскоп 3‑осевой | СКО начального смещения | `gyro_bias_stability_deg_hr` → 0.02 °/ч |
| БИНС | Гироскоп 3‑осевой | Белый шум (ARW) | `gyro_arw_deg_sqrt_hr` → 0.002 °/√ч |
| БИНС | Акселерометр 3‑осевой | СКО начального смещения | `accel_bias_stability_mg` → 0.02 мg |
| БИНС | Акселерометр 3‑осевой | Белый шум (VRW) | `accel_vrw_m_s_sqrt_hr` → 0.002 м/с/√ч |
| БИНС | (модель) | Random walk смещений | в коде: 0.1·σ_bias·√dt на шаг |
| ГНСС | Позиция PVT | Независимый гауссов шум по осям | `position_std_m` → 0.25 м |
| ГНСС | Скорость PVT | Независимый гауссов шум по осям | `velocity_std_m_s` → 0.02 м/с |
| ГНСС | Частота | Интервал между решениями | `update_rate_hz` → 20 Гц |

## Графики моделируемых величин от времени

После `poetry install` в каталоге `output/` появятся PNG:

```bash
poetry run python -m src.plot_instruments
```

| Файл | Содержание |
|------|------------|
| `ins_gyro_accel_vs_time.png` | Измеренные **ω** и **f** в связанной СК (после шума и смещений), каждый шаг интегрирования |
| `ins_biases_vs_time.png` | Смещения гиро и акселя во времени |
| `gnss_errors_vs_time.png` | Ошибка позиции и скорости ГНСС к истине на эпохах выдачи |

Параметры прогона: `simulation.dt_s`, `duration_s`, `default_seed` в JSON.
